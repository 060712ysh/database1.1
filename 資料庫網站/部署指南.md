# 🎓 資訊工程學系 師生入口網 - 部署指南

## 📋 快速開始

### 第一步：數據庫初始化

1. **確保 MySQL 服務已運行**
   - Windows: 啟動 MySQL 服務
   - 預設連接：localhost, 用戶名 root，密碼空

2. **訪問初始化腳本**
   - 打開瀏覽器，訪問：`http://localhost/資料庫網站/www/setup_db.php`
   - 點擊開始初始化
   - 完成後將自動導向首頁

### 第二步：系統登入

系統已創建了多個測試帳號供演示使用：

#### 🔓 管理員帳號
- **帳號**：`admin`
- **密碼**：`admin`
- **權限**：全系統管理、帳號管理、開課管理、預約審核、留言審核、檔案管理

#### 👨‍🏫 教師帳號
- **帳號**：`teacher_wang` / `teacher_lin` / `teacher_chen`
- **密碼**：`admin`
- **權限**：個人簡歷維護、教學大綱編寫、成績批次登錄

#### 👨‍🎓 學生帳號
- **帳號**：`student_001` / `student_002` / `student_003`
- **密碼**：`admin`
- **權限**：線上選課、課表查詢、教室預約、系統留言

#### 👤 訪客
- **無需登入**，即可瀏覽
- **可訪問**：師資陣容、實驗室介紹

## 📚 核心功能說明

### 1️⃣ 線上選課系統 (學生功能)
- **進入**：學生登入 → 線上選課
- **功能**：
  - 查看當學期全部課程
  - 即時顯示「已選人數 / 人數上限」
  - 加選 / 退選課程
  - 自動驗證容量與重複選課

### 2️⃣ 成績批次登錄 (教師功能)
- **進入**：教師登入 → 班級成績批次登錄
- **功能**：
  - 選擇授課課程
  - 系統自動載入該課程的修課名單
  - 輸入平時、期中、期末成績
  - 自動計算總成績
  - 批次保存

### 3️⃣ 帳號管理 (管理員功能)
- **進入**：管理員登入 → 帳號管理
- **功能**：
  - 新增學生 / 教師 / 管理員帳號
  - 設定登入帳號與密碼
  - 刪除帳號

### 4️⃣ 開課管理 (管理員功能)
- **進入**：管理員登入 → 開課管理
- **功能**：
  - 新增學期課程
  - 設定課程代碼、名稱、授課教師
  - 設定修課人數上限
  - 設定上課時間
  - 刪除課程

### 5️⃣ 預約審核 (管理員功能)
- **進入**：管理員登入 → 預約審核
- **功能**：
  - 查看學生教室預約申請
  - 核准或拒絕預約

### 6️⃣ 留言審核 (管理員功能)
- **進入**：管理員登入 → 留言審核
- **功能**：
  - 查看系統留言
  - 回覆學生留言

### 7️⃣ 檔案管理 (管理員功能)
- **進入**：管理員登入 → 檔案管理
- **功能**：
  - 上傳系統檔案
  - 下載檔案
  - 刪除檔案

### 8️⃣ 課表與成績查詢 (學生功能)
- **進入**：學生登入 → 課表與成績
- **功能**：
  - 查看當學期課表
  - 切換學期查詢歷年成績
  - 自動計算學期平均成績

### 9️⃣ 個人簡歷維護 (教師功能)
- **進入**：教師登入 → 個人簡歷維護
- **功能**：
  - 更新個人信息（姓名、職稱、電話、信箱）
  - 資料會自動同步至師資陣容頁面

## 🗄️ 數據庫架構簡述

### 核心表結構

| 表名 | 用途 | 重點欄位 |
|-----|------|--------|
| **Users** | 用戶帳號管理 | id, username, password_hash, role |
| **Teachers** | 教師資訊 | teacher_id, user_id, name, title, email, phone |
| **Students** | 學生資訊 | student_id, user_id, name, enrollment_year |
| **Courses** | 課程資訊 | course_id, course_code, course_name, semester, teacher_id, capacity, schedule |
| **Enrollments** | 選課與成績 | enrollment_id, student_id, course_id, assignment_scores, midterm_score, final_score, total_score |
| **Reservations** | 教室預約 | reservation_id, student_id, room_name, purpose, start_time, end_time, status |
| **Messages** | 系統留言 | message_id, sender_id, content, reply_content, created_at |

### 關鍵設計原則

1. **級聯刪除 (ON DELETE CASCADE)**
   - 刪除課程時自動清除相關選課紀錄
   - 刪除用戶時自動清除相關資料

2. **唯一性約束**
   - 學生選課時防止重複選擇：`UNIQUE KEY(student_id, course_id)`

3. **容量驗證**
   - Course 表的 `capacity` 欄位控制選課人數上限

### 系統工作流程

1. **用戶請求** → `index.php`
2. **檢查 Session** → 判斷身份 (Admin / Teacher / Student / Guest)
3. **載入側邊欄** → 根據身份顯示對應選單
4. **路由分派** → 根據 `?page=xxx` 參數載入相應模組
5. **模組處理** → 連接資料庫，執行 CRUD 操作
6. **回傳結果** → 顯示於右側內容區

## 📝 注意事項

### 安全性提示
- ⚠️ **生產環境**：請勿使用預設密碼，改用 `password_hash()` 加密
- ⚠️ **SQL 注入防護**：已使用 Prepared Statements 參數化查詢
- ⚠️ **Session 管理**：建議配置 Session 超時時間

### 性能優化
- 🚀 考慮在常查詢欄位上建立索引（如 `semester`、`student_id`）
- 🚀 大量選課操作時，建議使用事務控制 (Transaction)

### 功能擴展建議
- 📌 添加成績發佈通知功能
- 📌 實現退選時間限制
- 📌 添加教室衝突檢查
- 📌 實現在線簽到簽退
