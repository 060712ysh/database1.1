# 🎓 資訊工程學系 - 師生入口網與系務管理系統
> System Requirement Specification (SRS) & Architecture Document

## 壹、 專案概述
本系統專為資訊工程學系打造，採用**「整合式入口網 (Unified Portal)」**設計思維。全站以單一頁面為核心，透過登入身分動態解鎖專屬的左側選單與功能模組。系統涵蓋公開資訊展示、系辦行政管理、教師教學評量，以及學生自主選課等四大核心維度，實現校園資訊數位化與流程自動化。

## 參、 角色與權限定義表 (RBAC)

| 角色 (Role) | 適用對象 | 權限層級 | 核心職責與操作範圍 |
| :--- | :--- | :--- | :--- |
| **Guest** | 一般訪客 | 最低 (唯讀) | 僅能瀏覽全系公開之師資陣容、實驗室簡介與系統公告。 |
| **Student** | 資工系學生 | 一般 (個人) | 僅能操作綁定自身學號之選課、成績、預約與留言功能。 |
| **Teacher** | 授課教師 | 進階 (個人) | 僅能維護個人簡歷、上傳負責課程之大綱，並對修課學生進行成績登錄。不可建立帳號或修改他人資料。 |
| **Admin** | 系辦行政人員 | 最高 (全域) | 掌握系統底層資料，負責建立帳號、開立學期課程、審核全系資源借用，並具備覆寫/修改所有資料之代管權限。 |

## 肆、 核心功能模組說明

### 4.1 公開資訊模組 (Public)
* **師資陣容與簡歷**：列出全系教師基本資料、聯絡方式、教學歷程與學術榮譽。
* **實驗室與學術研究**：公開各實驗室研究方向、招生資訊及教師著作清單。
* **單一登入入口 (SSO)**：提供統一帳號密碼輸入框，後端自動判斷身分並分流權限。

### 4.2 系統管理員模組 (Admin)
* **系統帳號管理**：新增/刪除學生帳號，建立教師登入帳號並綁定對應之 `teacher_id`。
* **學期開課管理 (核心)**：新增當學期課程，設定課程代碼、名稱、授課教師、上課時間，以及「修課人數上限」。
* **預約單與留言審核**：檢視學生送出之教室預約單（核准/拒絕）；檢視全站留言並進行回覆。
* **系統檔案中心**：上傳系統通用表單、法規文件或更新教師大頭照等靜態資源。

### 4.3 教師功能模組 (Teacher)
* **個人簡歷與學術維護**：自主更新個人簡介、教學經歷、請益時間、研究計畫與論文清單。
* **課程大綱撰寫**：針對系辦指派給自己的課程，填寫並發布詳細之教學大綱與評分標準。
* **班級成績批次登錄 (核心)**：
  * **自動接收名單**：自動同步學生線上選課後之修課名單，無須手動加入。
  * **批次輸入介面**：提供表格式介面，供教師期末一次性快速輸入全班之平時、期中、期末成績。

### 4.4 學生專區模組 (Student)
* **線上自主選課系統 (核心)**：
  * **瀏覽與即時名額**：檢視當學期開立課程，即時顯示「已選人數 / 人數上限」。
  * **加退選機制**：名額內允許自由加選，具備重複選課防呆阻擋；允許於規定時間內退選。
* **課表與成績查詢**：預覽當學期個人課表，切換學期查詢歷年各科成績與平均。
* **教室預約與留言**：選擇可用之會議室填寫借用事由送出審核；發送系統文字留言給系辦或教師。

## 伍、 業務邏輯與資料流 (Data Flow)

📌 **課程與成績生命週期**
本系統的選課與成績管理遵循嚴謹的三階段資料流：

1. **【建立期】管理員開課**：由 Admin 於後台建立新學期課程（設定人數上限、綁定授課教師）。
2. **【選課期】學生自主選課**：學生登入選課系統，進行加/退選。系統底層即時 `INSERT/DELETE` `Enrollment` 資料表，並檢查 `Course` 表的 `capacity`（容量）。
3. **【評量期】教師批次打分**：選課結束後，Teacher 登入系統，直接撈出 `Enrollment` 中的名單，利用 Array 表單批次 `UPDATE` 全班成績。

## 陸、 資料庫設計重點 (Schema Highlights)

為了支撐權限隔離與線上選課邏輯，資料庫具備以下關鍵設計：

* **權限身分綁定 (Row-Level Security)**
  `Admin` 表透過 `teacher_id` 欄位建立外鍵關聯。當身分為 `Editor` 時，系統會藉此限制該帳號只能對自己的資料進行 `UPDATE` 或 `DELETE`。
* **學期與容量控管 (Course Table)**
  擴充 `Course` 資料表，加入 `semester` (學期，如：113-1) 與 `capacity` (修課人數上限) 欄位，做為學生加選時的驗證基準。
* **多對多選課關聯 (Enrollment Table)**
  作為 `Student` 與 `Course` 的橋接表，負責記錄學生的：`midterm_score` (期中)、`final_score` (期末)、`assignment_scores` (平時作業) 與 `total_score` (總成績)。
* **級聯刪除 (Cascade Constraints)**
  設定嚴謹的 `ON DELETE CASCADE`。若管理員刪除某門課程，該課程底下的所有選課紀錄 (`Enrollment`) 會自動清除，防止出現孤兒資料 (Orphan Data)。
